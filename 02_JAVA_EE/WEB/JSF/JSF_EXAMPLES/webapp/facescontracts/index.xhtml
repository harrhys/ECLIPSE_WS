Resource Library Contracts Introduction

An enhancement to the JSF resource library has made it easier to include new and/or additional templates for use within an application. The new enhancement enables multiple templates to be utilized within an application, with the ability to dynamically or statically define which template is to be applied to specified views.

JSF allows templates to be created by adding a folder named /contracts to the root of an applicationâ€™s web directory, or into the /META-INF directory. Once added, one or more template directories can be placed inside of the contracts folder, each containing an associated template file and any optional resource files (CSS, JavaScript, etc.) to be used along with the template. The template file must be named template.xhtml.

Description of the Example

This example demonstrates that a facelets template can come from a resource library contract. That means you can set different areas in your application to use a different resource library contract.

In this example, there are two templates, default contract and user contract, that are used in two areas, the default area and the user area. To see how a particular template is triggered, see the WEB-INF/faces-config.xml file.

Use This Example

You can click Default Area to go to the default area which uses the 'default' contract.

You can click User Area to go to the user area which uses the 'user' contract.
