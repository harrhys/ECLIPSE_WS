Faces Flow in JavaServer Faces 2.2

In JSF 2.2, a solution has been adopted for binding multiple JSF views to each other, allowing them to share information with each other. This solution is called faces flow, and it allows a group of interrelated views to belong to a flow instance, and information can be shared across all of the views belonging to that flow instance.

Description of the Example

This example demonstrates the use of the JavaServer Faces 2.2 faces flows feature.

In this example, a FlowScoped Managed Bean is used to set or get the value of the flow item in a sign-up process. The configuration detail can be found in the signup-flow.xml file.

In this scenario, an account registration system is simulated. On the first page, input your e-mail address and password to create a new WebLogic account. On the next page, fill in all required personal information to complete the registration. After submitting the information, all of the registration information is displayed on a confirmation page. After confirming your information, a 'Sign up successful' page is displayed.

In the registration process above, if you go back to the previous page using the Back button, your information is retained. However, if you return the index page from the 'Sign up successful' page, all of your registration information is lost.

Now you can start to register your own WebLogic account.
